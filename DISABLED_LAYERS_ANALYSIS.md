# تحليل الطبقات غير المفعلة في AmalSense

## الطبقات الرئيسية غير المفعلة بالكامل (6 طبقات)

### 1. Feedback Loop (حلقة التغذية الراجعة)
**الحالة:** موجود كفكرة لكن غير مفعل بالكامل

**المتطلبات:**
- المستخدم يعطي تقييم للتحليل
- يتم حفظ التقييم في قاعدة البيانات
- يرتبط بالتحليل نفسه

**التفعيل المطلوب:**
- جدول `feedback` في قاعدة البيانات
- `endpoint` لاستقبال التقييم
- ربط بـ `event_id` (معرف الحدث)

---

### 2. Self-Evaluation Layer (طبقة التقييم الذاتي)
**الحالة:** المنصة تحلل نفسها لكن لا تسأل نفسها

**الأسئلة المطلوبة:**
- هل كان التحليل جيد؟
- هل البيانات كافية؟
- هل الثقة منخفضة؟

**المخرجات المطلوبة:**
- `quality_score` - درجة الجودة
- `data_quality` - جودة البيانات
- `analysis_depth` - عمق التحليل

---

### 3. Meta-Learning (التعلم الذاتي)
**الحالة:** أعمق وأغلى من طبقة التقييم الذاتي، غير مفعلة فعلياً

**المتطلبات:**
- النظام يراقب نفسه
- متى يفشل؟
- متى تكون الثقة منخفضة؟
- متى تتناقض النتائج؟

**التفعيل:**
- البيانات تُخزن عبر الزمن
- تحليل الأداء عبر الزمن

---

### 4. Weekly Self Report (التقرير الأسبوعي)
**الحالة:** موجود في التصميم لكن غير مفعل

**المتطلبات:**
- `cron job` أسبوعي
- عدد التحليلات
- متوسط الثقة
- أكثر مصادر البيانات
- أكثر المشاعر ظهوراً

---

### 5. Survey Calibration Layer (طبقة معايرة الاستطلاع)
**الحالة:** فكرة ممتازة لكن غير موجودة تقريباً

**المتطلبات:**
- استبيان بسيط
- تخزين النتائج
- تعديل المؤشرات بناءً عليه

---

### 6. Role-Based Cognitive Interface (واجهة معرفية قائمة على الأدوار) - جزئياً فقط
**الحالة:** المنصة تعرض نفس التحليل تقريباً للجميع

**المتطلبات:**
- إذا السؤال اقتصادي → يبرز مؤشرات اقتصادية
- إذا إعلامي → يبرز الرسد والسبب

## الطبقات المفعلة جزئياً فقط

### Context Layer
**الحالة:** التحليل يعتمد على النص نفسه وبعض الوزن الرمزي

**المتطلبات الناقصة:**
- ربط البحث بسياق جغرافي
- ربط بتاريخ أحداث مشابهة
- فهم نوع الحدث (اقتصادي، سياسي، اجتماعي)

---

### Explainability Layer
**الحالة:** توجد تفسير لكن لا يزال قريب من السرد

**الأسئلة المطلوبة:**
- لماذا أعطى النظام هذه النتيجة؟
- ما العامل الأول؟
- ما البيانات التي أثرت أكثر؟
